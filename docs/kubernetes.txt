Kubernetes Fundamentals

Kubernetes is an open-source container orchestration platform that automates the deployment, scaling, and management of containerized applications. Originally developed at Google and donated to the Cloud Native Computing Foundation, it has become the de facto standard for running workloads across clusters of machines.

The smallest deployable unit in Kubernetes is the pod. A pod wraps one or more containers that share the same network namespace and storage volumes, meaning containers in the same pod can talk to each other over localhost. In practice most pods contain a single application container, sometimes accompanied by a sidecar container for logging or networking.

Pods are rarely created directly. Instead, a Deployment describes the desired state: which container image to run, how many replicas to maintain, and how updates should roll out. The Deployment controller continuously compares the actual state of the cluster with the desired state and creates or removes pods to close the gap. This reconciliation loop is the central design principle of Kubernetes: you declare what you want, and controllers work to make it true.

Because pods are ephemeral and their IP addresses change when they are rescheduled, Kubernetes provides Services to give a stable network identity to a set of pods. A ClusterIP service exposes pods inside the cluster, a NodePort service opens a port on every node, and a LoadBalancer service provisions an external load balancer in cloud environments. Service discovery works through cluster DNS, so an application can reach another service simply by its name.

Configuration is separated from code using ConfigMaps for plain settings and Secrets for sensitive values like passwords and API keys. Both can be mounted as files or injected as environment variables.

The cluster itself consists of a control plane and worker nodes. The control plane runs the API server, the scheduler that assigns pods to nodes, the controller manager, and etcd, the distributed key-value store that holds all cluster state. Each worker node runs the kubelet, which starts and monitors containers, and a network proxy. When you run a kubectl command, you are talking to the API server, which validates the request and persists the new desired state to etcd before controllers act on it.

For machine learning platforms, Kubernetes is the foundation on which tools like Kubeflow and KServe are built, providing the scheduling, scaling, and isolation needed to run training jobs and model servers reliably.
