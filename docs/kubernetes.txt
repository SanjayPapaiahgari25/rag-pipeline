Kubernetes

Kubernetes is an open-source platform for automating the deployment, scaling, and management of containerized applications. It groups one or more containers into a unit called a pod, which is the smallest deployable object. Containers in the same pod share networking and can share storage, and they are always scheduled together onto the same node.

A Kubernetes cluster has a control plane and a set of worker nodes. The control plane holds the desired state of the system and continuously works to make reality match it. The API server is the front door through which all commands flow. The scheduler decides which node each new pod should run on based on resource requests and constraints. Controllers watch the cluster and take corrective action, for example creating a replacement pod when one dies. The cluster's state is stored in etcd, a consistent key-value store.

The central design principle is declarative configuration. Rather than issuing imperative commands, a user submits a manifest describing the desired state, and Kubernetes reconciles toward it. A Deployment, for instance, declares how many replicas of a pod should exist; if a pod crashes, the controller notices the gap and starts a new one to restore the count.

Because pods are ephemeral and their network addresses change, a Service provides a stable virtual endpoint and load-balances traffic across the matching pods. This lets other components address a logical service without tracking individual pod addresses.

Kubernetes also manages scaling. The Horizontal Pod Autoscaler adds or removes pod replicas based on observed metrics such as CPU utilization, and the cluster autoscaler can add nodes when there is no room to schedule pending pods. Resource requests and limits let the scheduler pack workloads efficiently while protecting nodes from overcommitment. This combination of self-healing, declarative state, and automated scaling is why Kubernetes became the standard substrate for running services and machine learning workloads at scale.
